// design criteria

Three hard requirements

ARC ships when all three are met. Each one is binary — either it works or it doesn't.

⚡
01 — Speed
Under 5 minutes
Quality Claude output on any new project in under five minutes. /arc-init handles setup. /arc-kickoff fills context.
◈
02 — Consistency
Uniform quality
The same high output bar regardless of project type, domain, team size, or how early in the day you're reading the output.
∅
03 — Memory
Zero re-explanation
Preferences, conventions, and constraints live in files — not in your head. Claude reads them once. You never repeat yourself.

// slash commands

Six operational skills

ARC's runtime layer. Called from Claude Code as slash commands. Each has a precise trigger and expected time cost.

Setup phase
/arc-init
Six-question wizard. Generates a complete CLAUDE.md from scratch — picks overlays, domain modules, and quality level automatically.
~2 min new project
Setup phase
/arc-kickoff
Deep project interview. Surfaces implicit requirements — tech constraints, team norms, quality bar, risk tolerance — before the first line of work.
15–30 min after init
Ongoing phase
/arc-resume
Daily session start. Loads project state, outstanding todos, and last context. Claude continues where it left off with no re-briefing.
<1 min every session
Ongoing phase
/arc-rekickoff
Mid-project pivot. Re-runs the kickoff interview when scope changes, tech stack shifts, or the project direction changes meaningfully.
10–20 min scope pivot
Quality phase
/arc-check
Output quality gate. Three-level check: existence, substantive, wired. Domain-specific checks including the 21-item security surface scan.
2–5 min any artifact
Quality phase
/arc-progress
Project dashboard. Renders current state — completed todos, open decisions, lessons learned, and blockers — as a single structured view.
~1 min status check

// core/

Core principle layer

Six foundational guides that underpin every template and pattern. Model-agnostic — they work because of how language models process context, not Claude quirks.

core/principles.md
Specificity Amplification
Generic input → generic output. Every detail you add eliminates a category of wrong answers. The root cause of ~60% of bad Claude output.
core/anti-slop.md
Anti-Slop System
Structured rules against the AI generic-output trap: delete test, hedge scan, filler audit, preamble ban, and constraint injection.
core/mental-models.md
Mental Models
How to think about prompting: context layers, the solution space model, and why Claude is a very smart peer — not a search engine.
core/execution-discipline.md
Execution Discipline
Plan-first rule, elegance gate, verification protocol. What must happen between receiving a task and declaring it complete.
core/failure-modes.md
Failure Mode Catalog
Named failure patterns — sycophancy spiral, scope creep, stub delivery, unverified assumption — with recovery protocols for each.
core/prompt-debugging.md
Prompt Debugging
Systematic "my prompt isn't working" troubleshooter. Fourteen diagnostic questions organized by failure symptom, not by prompt structure.
