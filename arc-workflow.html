Skills Reference
11 Skills, Three Groups
Run by Claude Code in your project directory
Setup & Context
/arc-init Start of any new project. Creates CLAUDE.md + tasks/. Takes 2 minutes.
/arc-kickoff Deep context: 15-min interview, project brief, arch diagram, portable export.
/arc-resume Start of every session. Reads STATE.md, surfaces blockers, uncommitted changes, open PRs.
/arc-rekickoff Mid-project pivot or scope change. Re-orients context without a full restart.
/arc-export Portable context prompt. Paste into Claude.ai or share with teammates.
Planning & Execution
/arc-plan Before writing code. Default EXPAND: finds the real product, maps delight opportunities, failure modes, reversibility index.
/arc-review Before creating a PR. Two-pass diff: CRITICAL (security, auth, IDOR) then INFORMATIONAL (quality, wiring, tests).
/arc-ship Fully automated: merge → tests → review → version bump → changelog → commits → push → PR.
Quality & Retro
/arc-check After any significant output. Stubs, wiring, specificity, deployment safety. Domain-specific checks.
/arc-progress Project health dashboard. Context health, open questions count, todo summary.
/arc-retro End of sprint. Reviews what shipped, what broke. Extracts lessons. Appends to tasks/lessons.md.

Three Non-Negotiables
1
Plan before building
Non-trivial tasks start with a written plan in tasks/plan.md. Prevents the most common Claude failure: implementing before understanding the problem.
2
Anti-slop enforced
Every output must contain project-specific information. If it could apply to any project in the same category, it's generic — it gets rejected and redone.
3
Evidence before done
Code needs clean diagnostics. Builds need exit code 0. Plans need verification steps. Nothing is marked complete without proof.
