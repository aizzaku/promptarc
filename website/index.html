How it works

Four steps. Then you just work.

ARC does one thing upfront: it teaches Claude about your project. Every request after that gets a targeted answer — because Claude already knows your stack, quality bar, and constraints.

01
$ npx arc-setup
Install
Once · ~10 seconds

Copies ARC's templates and principles into your Claude Code project. Run it once and forget it.

02
/arc-init
Initialize
Once per project · ~1 min

Pick your domain and quality bar. ARC writes a CLAUDE.md tailored to your project — stack, constraints, and anti-patterns included.

03
/arc-kickoff
Interview (optional)
Optional · ~15 min

8 questions. Claude learns your voice, tech decisions, and success criteria. Creates a project brief saved to your tasks folder.

04
ask Claude anything
Work
Every session · forever

Type short, direct requests. Claude answers with full context — no clarifying questions, no generic output.

See the difference

Same request. No clarifying questions.

You type the same thing either way. ARC pre-loads Claude with your project context — so instead of asking five questions, Claude answers the one you asked.

From the framework

Two things most people skip.

The ARC framework runs on eight principles. These two eliminate the most failure modes.

01 / 08
Specificity Amplification

Generic input produces generic output. Always. Every detail you add eliminates an entire category of wrong answers — because it narrows the space of likely responses Claude can produce.

Without
"write good code"
With
"TypeScript function validating email per RFC 5322, returns Valid | Invalid<reason>, handles plus addressing and international domains"
07 / 08
Negative Examples

Showing Claude what you don't want is more instructive than describing what you do. There are infinite ways to be "clear" — but you can enumerate the specific failure modes you're trying to avoid.

Instead of "write clearly," use:
DO NOT: - Start with "In today's fast-paced world" - Use bullet points as default structure - Summarize what was just said - End with "In conclusion" or "To summarize"
