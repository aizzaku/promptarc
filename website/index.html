The commands

11 skills. Full cycle.

Setup, plan, build, review, ship, retro — one coherent loop. The daily minimum is /arc-resume → work → /arc-check.

Setup phase
/arc-init

Pick domain and quality bar. Writes CLAUDE.md and initializes STATE.md in under a minute. Run once per project.

~2 min new project
Setup phase
/arc-kickoff

8-question interview. Calibrates voice, captures constraints, validates CLAUDE.md with a synthetic test, and closes any gaps it finds.

15–30 min after init
When pivoting
/arc-rekickoff

Re-orient after scope changes. Targeted Q&A on what's changed, then updates CLAUDE.md, task files, and STATE.md.

~10 min scope change
Ongoing
/arc-resume

Start every session. Reads STATE.md and orients in two sentences — last session, current focus, what's next. No re-explaining the project.

<1 min every session
Quality
/arc-check

Goal-backward output verification. Detects stubs, wiring failures, and domain-specific anti-patterns. Produces a structured gap list.

~2 min anytime
Quality
/arc-progress

Project health dashboard. Context completeness, open decisions, todo summary, and exactly one recommended next action.

~1 min anytime
Planning
/arc-plan

Before writing code. Default EXPAND mode: finds the real product hiding in the request, maps delight opportunities, failure modes, and reversibility. Also: HOLD and REDUCE modes.

~10 min before coding
Pre-PR
/arc-review

Two-pass diff-based code review. Pass 1 CRITICAL: security, auth, IDOR, SQL injection. Pass 2 INFORMATIONAL: quality, tests, wiring. Domain-aware. Never commits or pushes.

~3 min before PR
Ship
/arc-ship

Fully automated deployment pipeline. Tests → review → version bump → changelog → logical commits → push → PR creation. No confirmation unless CRITICAL issues are found.

~5 min ready to land
Retrospective
/arc-retro

End-of-sprint retrospective. Reviews what shipped, what broke, what was learned. Appends lessons to tasks/lessons.md. Checks context health — recommends rekickoff if stale.

~5 min sprint close
Export
/arc-export

Assembles a portable context prompt from CLAUDE.md, brief, and STATE.md. Paste into Claude.ai, share with teammates, or use as a handoff document.

<1 min anytime

Architecture

Layered composition

A CLAUDE.md is assembled from independent layers. Each layer adds without overwriting. Upper layers take precedence.

base.md
+ [task-type overlay]
+ [industry overlay]
+ [domain module]
+ [project overrides]
─────────────────────
= CLAUDE.md (< 3,500 words)
project-overrides.md yours
Your rules for this specific project
domains/fintech.md industry
Domain vocabulary, pitfalls, stack context
claude-md/saas.md overlay
NON-NEGOTIABLE rules, defaults, voice
claude-md/software-eng.md task
Work-type behavior and code standards
base.md foundation
Universal identity, tone, anti-slop rules
