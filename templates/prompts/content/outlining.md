# Pattern: Structure an Argument

> "I want to structure my content before writing"

**Principles used**: Chain-of-Thought Scaffolding, Constraint-Driven Creativity

---

## When to Use
- You have an angle and research but need to structure the argument
- Draft feels disorganized — need to find the throughline
- Planning a longer piece that needs intentional pacing

## The Pattern

```
My thesis: {{SPECIFIC_CLAIM}}
My audience: {{WHO}} — they currently believe {{CURRENT_ASSUMPTION}}
My goal: shift them to believing {{NEW_UNDERSTANDING}}

Key evidence I have:
- {{EVIDENCE_1}}
- {{EVIDENCE_2}}
- {{EVIDENCE_3}}

Build an outline that:
1. Opens with a hook that creates tension or curiosity (NOT a definition or background)
2. Establishes why the current assumption is wrong or incomplete
3. Presents the new understanding with evidence
4. Addresses the strongest counter-argument
5. Closes with an insight or implication (NOT a summary of what was said)

For each section, give me:
- The purpose (what does this section DO for the reader?)
- The key point (one sentence)
- The evidence or example that carries this section

The outline should feel like an argument with momentum, not a list of topics.
```

## Why It Works
Framing the outline as "shift the reader from X to Y" gives it narrative direction instead of being a topic list. Requiring each section to have a purpose prevents filler sections.

## Common Mistakes
- Outlining by topic ("Introduction, Background, Analysis, Conclusion") → creates a generic structure
- Not identifying the reader's current belief → no tension, no reason to keep reading
- Including a "Background" section → usually filler. Weave context into the argument.

## Variations
- **Video script outline**: Add "Include a re-hook at the 30% mark where viewers typically drop off."
- **Thread outline**: "Each section becomes one post in a thread. First post is the hook. Last post is the CTA. Each post must work standalone AND advance the thread."
