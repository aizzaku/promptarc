# Pattern: Create a Pitch Deck Narrative

> "I want to create a pitch deck narrative"

**Principles used**: Context Layering, Constraint-Driven Creativity, Negative Examples

---

## When to Use
- Building a pitch deck for investors, partners, or internal stakeholders
- Need the narrative arc before designing slides
- Have all the content but can't find the story

## The Pattern

```
Build the narrative for a pitch deck for: {{COMPANY/PRODUCT}}

Stage: {{STAGE — pre-seed, seed, Series A, etc.}}
Audience: {{WHO_SEES_THIS — VCs, angels, corporate partners, internal leadership}}
Ask: {{WHAT_YOU_WANT — funding amount, partnership, approval, etc.}}

Key facts:
- Problem: {{THE_PROBLEM_YOU_SOLVE}}
- Solution: {{YOUR_SOLUTION}}
- Traction: {{WHAT_YOU_HAVE — users, revenue, growth rate, LOIs, pilots}}
- Market: {{SIZE_AND_SEGMENT}}
- Team: {{WHY_THIS_TEAM}}
- Moat: {{COMPETITIVE_ADVANTAGE}}

Build the deck narrative (not slides — the STORY):
1. The opening: What's the tension? (A big problem, a missed opportunity, a broken status quo)
2. The reveal: How does your solution change the game? (Demo moment, not feature list)
3. The proof: Why should they believe you? (Traction, team credentials, market timing)
4. The vision: Where does this go? (Market size, expansion path, endgame)
5. The ask: What do you need and what will you do with it? (Specific milestones the funding enables)

For each narrative beat:
- The core message (one sentence)
- The supporting evidence
- The slide type that carries this (data slide, demo screenshot, quote, chart)

Do NOT:
- Start with "Company X is a platform that..." — start with the problem
- List features — show outcomes
- Claim "no competitors" — show why your approach wins
- Use hockey-stick projections without justifying the inflection points
```

## Why It Works
Treating the deck as a narrative arc (tension → reveal → proof → vision → ask) creates momentum. Working at the story level before the slide level prevents decks that are feature lists masquerading as pitches.

## Common Mistakes
- Building slides before narrative → deck is a data dump, not a story
- Leading with the solution → audience doesn't care until they feel the problem
- "We have no competitors" → immediate credibility loss with sophisticated audiences
