# Pattern: Design a System, Habit, or Routine

> "I want to design a system/habit/routine"

**Principles used**: Context Layering, Constraint-Driven Creativity, Specificity Amplification

---

## When to Use
- Building a new habit that needs to stick
- Designing a personal system (morning routine, weekly review, filing system)
- Current routine isn't working and needs a redesign

## The Pattern

```
I want to build a {{HABIT | ROUTINE | SYSTEM}} for: {{WHAT}}

Current state: {{WHAT_I_DO_NOW — including "nothing"}}
Previous attempts: {{WHAT_I'VE_TRIED_AND_WHY_IT_FAILED}}
Energy/willpower: {{WHEN_IS_MY_ENERGY_HIGHEST? WHEN_IS_IT_LOWEST?}}
Non-negotiable constraints: {{THINGS_I_CAN'T_CHANGE — work schedule, family obligations, etc.}}

Design the system:
1. The trigger: What existing habit or event does this attach to? (No willpower-dependent triggers like "when I feel like it.")
2. The minimum version: What's the absolute smallest version of this that still counts? (e.g., "open the notebook" not "journal for 30 minutes")
3. The full version: What does the ideal version look like when the habit is established?
4. The ramp-up: How do I get from minimum to full? (Timeline with specific milestones)
5. The failure recovery: When I miss {{1 day | 1 week}}, what's the restart protocol? (Not "recommit" — a specific action.)
6. The measurement: How do I know it's working? (Leading indicator, not just "do I feel better")

Stress-test:
- Does this work on my worst day (low energy, bad mood, behind on work)?
- Does this survive travel or schedule disruption?
- Is the minimum version genuinely achievable in under 5 minutes?

Do NOT: Design the ideal version for day 1. Start embarrassingly small. Consistency beats ambition.
```

## Why It Works
Anchoring to existing triggers removes willpower dependence. The minimum viable version eliminates the "I don't have time" excuse. Explicit failure recovery prevents the "I already ruined my streak" spiral.

## Common Mistakes
- Starting with the full version → burnout within 2 weeks
- Willpower-dependent triggers ("every morning") → fails on first bad morning
- No measurement → can't tell if the system is actually producing results
