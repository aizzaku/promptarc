# Pattern: Analyze Something Without a Pre-Built Framework

> "I want to analyze something I don't have a framework for"

**Principles used**: Chain-of-Thought Scaffolding, Specificity Amplification

---

## When to Use
- Need to analyze something that doesn't fit a standard framework
- The subject is novel or cross-disciplinary
- Standard frameworks feel forced for this situation

## The Pattern

```
I need to analyze: {{SUBJECT}}

Purpose: {{WHAT_DECISION_OR_UNDERSTANDING_THIS_SERVES}}
What I already know: {{EXISTING_CONTEXT}}

Don't apply a pre-built framework. Instead:
1. What are the dimensions that matter for this analysis? (Derive them from the purpose, not from a textbook.)
2. For each dimension: What's the current state? What's the evidence?
3. What patterns emerge across dimensions? (What do you see when you look at the whole picture?)
4. What's the so-what? (Given this analysis, what should I do, think, or investigate further?)

Keep it focused: This analysis should be complete in under {{LENGTH — e.g., "2 pages" / "500 words"}}. If it needs to be longer, the analysis isn't focused enough.
```
