# Design / UX Kickoff Checklist

Run after universal checklist Phase 1. Ask 3 questions at a time.

---

## Required

### Deliverable & Scope

1. **What type of design work?**
   - UI design (screens, components, flows)
   - UX research (user interviews, usability testing, synthesis)
   - Design system (components, tokens, documentation)
   - Prototyping (low-fi wireframes / high-fi interactive)
   - Service design / journey mapping
   - Mixed

2. **What platform(s)?**
   - Web (desktop-first / responsive / mobile-first)
   - iOS native
   - Android native
   - Cross-platform mobile
   - Desktop app
   - Multiple

3. **What's the primary tool?**
   - Figma
   - Sketch
   - Adobe XD
   - Code-based (React + Storybook, etc.)
   - No preference / not decided

---

### Context & Constraints

4. **Does a design system exist?**
   - No — starting from scratch
   - Partial — some components/tokens defined
   - Yes — full system, must follow it strictly
   - Yes — but it needs to be extended

5. **Are brand guidelines defined?**
   - Yes — link or describe them
   - Partially — some things locked (logo, colors), others flexible
   - No — brand definition is part of this work

6. **What's the accessibility requirement?**
   - WCAG 2.1 AA (standard)
   - WCAG 2.1 AAA (high bar)
   - Section 508 (US federal)
   - None specified
   - Unknown — needs to be determined

---

### Handoff & Collaboration

7. **Is there a developer handoff? What does it look like?**
   - Yes — Figma to developer with annotations and specs
   - Yes — design produces code (React components, etc.)
   - No handoff — design is for internal decision-making only
   - TBD

8. **Who reviews and approves design decisions?**
   (Product manager, engineering lead, CEO, client, etc.)

9. **What existing user research or data exists?**
   - User interviews already conducted
   - Analytics / usage data available
   - No research yet — research is part of this work
   - None — designing from assumptions

---

## Conditional

### If UX research is in scope

10. **What research methods are available?**
    - User interviews (moderated)
    - Unmoderated usability testing
    - Surveys
    - Analytics review
    - Card sorting / tree testing

11. **Who are the participants?** (Not "users" — who specifically will be interviewed/tested?)

12. **What decisions will this research inform?** (The research must be tied to a concrete decision, not general understanding.)

---

### If design system work is in scope

13. **What's the target output?** (Figma component library / Storybook / Design tokens / All three)

14. **What's the adoption model?** (Teams adopt voluntarily / Mandated / Gradual migration from existing system)

15. **What's the governance model?** (Who maintains it? Who approves new components?)

---

### If starting from scratch (no existing design system/brand)

16. **What's the aesthetic direction?** (Words, references, examples of products you admire or want to avoid)

17. **What should it NOT look like?** (Anti-examples are often more useful than examples)
